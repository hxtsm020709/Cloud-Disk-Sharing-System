# 网盘共享系统 — 开发计划

> 版本：v1.0  
> 更新日期：2026-06-04

---

## 总体策略

- **分阶段推进**：每阶段完成后验证，确认可用再进入下一阶段
- **先骨架后血肉**：先搭建项目框架，再逐步填充业务功能
- **核心难点前置**：优先攻克百度登录协议分析，避免后期发现不可行
- **每阶段一测试**：确保当前阶段功能正常工作

---

## 阶段一：项目脚手架搭建

> 目标：搭好项目骨架，能启动、能访问

**后端：**
- [ ] 初始化 Node.js 项目（package.json、依赖安装）
- [ ] Express 基础服务器搭建
- [ ] SQLite 数据库初始化（建表）
- [ ] 基础配置（config.js、.env）
- [ ] 加密工具模块
- [ ] 管理员认证中间件

**前端：**
- [ ] 全局样式（style.css，淡蓝色主题）
- [ ] 管理员登录页（纯静态 HTML + CSS）

**验证标准**：`npm start` 启动成功，浏览器能打开登录页

---

## 阶段二：管理端 — 管理员认证 + 账号管理

> 目标：管理员能登录后台、添加和管理账号

- [ ] 管理员注册/登录逻辑
- [ ] Session 认证
- [ ] 后台首页（仪表盘），展示统计概览
- [ ] 账号列表页（分页、搜索）
- [ ] 添加账号页：
  - [ ] 手动导入表单（昵称、Cookie、会员信息）
  - [ ] 扫码提取功能（需要先完成阶段四的百度协议分析）
- [ ] 单个账号详情页（编辑、删除、查看使用记录）
- [ ] Cookie 加密存储与解密读取

**验证标准**：能登录后台、能添加/编辑/删除账号

---

## 阶段三：百度登录协议分析（核心技术难点）

> 目标：理解百度网盘扫码登录的完整流程，并能在代码中复现

- [ ] 抓包分析百度网盘 PC 端获取二维码的 API
- [ ] 抓包分析百度网盘手机 App 扫码确认的 API
- [ ] 分析请求参数、Headers、签名逻辑
- [ ] 编写 `baidu-auth.js` 服务：
  - [ ] `getQRCode()` — 获取登录二维码数据
  - [ ] `pollQRStatus(sign)` — 轮询二维码状态
  - [ ] `confirmLogin(sign, cookies)` — 用 Cookie 确认登录
  - [ ] `checkCookieValid(cookies)` — 检测 Cookie 有效性
- [ ] 编写单元测试验证

**验证标准**：能在代码中用已有 Cookie 完成一次完整的扫码登录流程

---

## 阶段四：管理员扫码提取 Cookie

> 目标：管理员在后台可以通过让手机扫描二维码的方式提取 Cookie

- [ ] 复用阶段三的二维码获取逻辑
- [ ] 扫码提取页面：展示二维码 → 管理员用百度 App 扫 → 拦截确认请求提取 Cookie
- [ ] 自动解析并保存提取到的 Cookie
- [ ] （备选方案）浏览器 Cookie 复制粘贴导入

**验证标准**：能通过扫码流程将一个新账号的 Cookie 保存到数据库

---

## 阶段五：分享链接 + 员工扫码端

> 目标：管理员能生成分享链接，员工能用链接完成扫码登录

- [ ] 分享链接管理页：
  - [ ] 生成链接（选择账号、选择有效期）
  - [ ] 链接列表（状态、使用次数）
  - [ ] 手动停用链接
- [ ] 员工扫码登录页（手机端）：
  - [ ] 调用手机摄像头
  - [ ] jsQR 实时解析二维码
  - [ ] 解析成功后将数据发回服务器
  - [ ] 服务器调用百度确认登录 API
  - [ ] 显示成功/失败结果
- [ ] 链接过期自动失效
- [ ] 使用日志记录

**验证标准**：生成链接 → 手机打开扫码 → PC端登录成功

---

## 阶段六：自动化检测 + 到期提醒

> 目标：系统自动检测 Cookie 状态和会员到期情况

- [ ] Cookie 有效性定时检测（node-cron）
- [ ] 会员到期时间提取和展示
- [ ] 后台首页到期提醒列表
- [ ] 异常状态邮件/页面通知

**验证标准**：后台首页能看到哪些账号即将到期，Cookie 状态自动更新

---

## 阶段七：本地部署 + 手机扫码 HTTPS 解决

> 目标：在本机部署完整可用的系统

- [ ] mkcert 生成局域网 HTTPS 证书
- [ ] 配置 Express HTTPS
- [ ] 手机通过局域网 IP 访问测试
- [ ] 整体流程端到端测试

**验证标准**：手机能通过局域网 HTTPS 打开扫码页面，完整流程可走通

---
